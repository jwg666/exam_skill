# “题海拾贝” 产品需求文档 (PRD)

## 1. 产品概述
**产品名称**：题海拾贝
**产品定位**：一款轻量级、移动端优先的智能刷题与知识学习应用。
**核心价值**：帮助用户利用碎片化时间进行知识测试、技能考核与趣味问答，通过数据统计、错题本与游戏化机制（打卡、成就、等级）提升用户的学习粘性与效率。

## 3. 核心功能详述

### 3.1 账号体系
- **登录/注册**：目前采用本地模拟的手机号+密码登录。注册需校验昵称、手机号（11位）、密码一致性（>=6位）。
- **鉴权拦截**：所有非登录页在加载时需检查用户状态（`checkLoginAuth`），未登录自动重定向至 `login.html`。

### 3.2 答题引擎（核心）
- **出题策略**：支持全部题目顺序出题、错题重练、随机抽题、单题查看。
- **状态保留**：答题过程中记录耗时（`elapsed`）、用户选项、标记状态（Flag）。
- **即时反馈**：用户点击选项后，立即判断对错并锁定选项，同时展示底部解析（Explanation）。
- **答题卡面板**：直观展示题目总览，标记“已答”、“未答”、“疑点标记”，支持点击题号快速跳转。
- **结算逻辑**：计算正确率，自动将做错的题目加入“错题本”，更新对应题库的完成进度，并在 `result.html` 展示评分。

### 3.3 学习反馈闭环
- **错题本闭环**：错题自动收录 -> 错题重练 -> 掌握后手动移出错题本。
- **成就闭环**：设置阶梯性目标（如：首次登录、累计50题、连续3天打卡、100%正确率等），触发时全局 Toast 提示并点亮徽章。

## 4. 数据结构设计 (目前基于 LocalStorage)
- **静态资源 (`questionBanks`)**：包含各题库的基础信息及题目数组（题目、选项、正确索引、解析）。
- **用户状态 (`appState`)**：
  - `user`: 昵称、手机号
  - `totalAnswered`, `totalCorrect`, `streak`, `checkedDays`: 全局学习数据
  - `wrongBook`, `favorites`, `history`: 数组集合
  - `bankProgress`: 记录各题库刷到了第几题
  - `achievements`: 已解锁成就ID集合
- **当前答题状态 (`quizState`)**：临时存储当前测验的上下文（题库ID、题目集、当前题号、用户答案数组、标记数组、耗时）。

## 5. 后续迭代规划探讨 (供讨论)
1. **后端接口接入**：将 LocalStorage 改造为真正的 RESTful API 或 GraphQL，引入真实数据库（MySQL/PostgreSQL）。
2. **题型扩展**：目前仅支持单选题，后续可扩展多选题、判断题、填空题。
3. **社交属性**：好友系统、真实排行、成就分享、错题讨论区。
4. **深色模式**：完善 `toggleDarkMode` 的实际 CSS 变量切换逻辑。
5. **性能与工程化**：引入前端框架（Vue/React/构建工具）进行组件化重构。
